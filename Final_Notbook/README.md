# ST422 Brief 8 — Road Safety Analysis

Analysis of DfT STATS19 road casualty data (2014–2025) for ST422 Statistical
Consulting. This repo contains the full data preparation and analysis pipeline
for Brief 8: identifying road safety hotspots, contributory factors, and
evaluating whether recent policy changes are working.

> **Note: the choropleth map (Section 5 of the analysis notebook) requires an
> active internet connection to fetch ONS boundary data from the ArcGIS REST API.**

---

## Folder Structure

```
ST422/
├── ST422_DataPrep.ipynb         ← Step 1: data cleaning and merging
├── ST422_Analysis_v2.ipynb      ← Step 2: full analysis pipeline
├── requirements.txt             ← Python dependencies
├── README.md                    ← this file
├── Data/                        ← raw DfT STATS19 CSVs (see below)
├── Cleaned/                     ← produced by ST422_DataPrep.ipynb
└── outputs/                     ← figures and tables produced by ST422_Analysis_v2.ipynb
```

> `Data/` and `Cleaned/` are not committed to Git — files are too large.
> `outputs/` figures and tables are regenerated by running the notebooks.

---

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Download the raw data files

Place the following files inside the `Data/` folder. Use the exact DfT
filenames — do not rename them.

```
dft-road-casualty-statistics-collision-1979-latest-published-year.csv
dft-road-casualty-statistics-casualty-1979-latest-published-year.csv
dft-road-casualty-statistics-vehicle-1979-latest-published-year.csv
dft-road-casualty-statistics-collision-provisional-2025.csv
dft-road-casualty-statistics-casualty-provisional-2025.csv
dft-road-casualty-statistics-vehicle-provisional-2025.csv
local-authority-ons-district-names.csv
```

Download from:
https://www.data.gov.uk/dataset/cb7ae6f0-4be6-4935-9277-47e5ce24a11f/road-safety-data

The `local-authority-ons-district-names.csv` lookup file is included in the
repo under `Data/`.

### 3. Paths

No path editing is required. Both notebooks automatically detect their
location using `os.getcwd()` and build all paths relative to the project
root. As long as the folder structure above is in place, the notebooks
will run on any machine without changes.

## How to Run

**Run in this order. Restart the kernel before each notebook.**

### Step 1 — Data preparation

Open and run `ST422_DataPrep.ipynb` top to bottom.

This produces four cleaned CSV files in `Cleaned/`:

| File | Description |
|---|---|
| `collisions_clean.csv` | One row per collision, 2014–2025 |
| `casualties_clean.csv` | One row per casualty |
| `vehicles_clean.csv` | One row per vehicle |
| `cas_full.csv` | One row per casualty with vehicle and collision context joined |

Open and run `ST422_Analysis_v2.ipynb` top to bottom.

## Reproducibility Notes

- Always restart the kernel before a full re-run
- Run Step 1 before Step 2 — the analysis notebook reads `cas_full.csv`
  produced by the data prep notebook
- All paths are relative to the project root — no local path editing needed
- No random sampling is used except in the bootstrap CI cell (seed set to 42)
- `collision_index` is loaded as string throughout to preserve leading zeros
- The choropleth map (Section 5) requires an active internet connection to
  fetch ONS boundary data from the ArcGIS REST API at runtime; all other
  sections run fully offline

## Year Window

The analysis covers 2014–2024 (confirmed) and January–June 2025 (provisional).
COVID years (2020–2021) are retained in the cleaned data but excluded from
trend fitting and flagged throughout. To change the year window, edit the
config cell at the top of `ST422_DataPrep.ipynb`.
