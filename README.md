# ST422 Team 1 — Brief 8: Road Safety Analysis

Analysis of DfT STATS19 road casualty data for ST422 Statistical Consulting.
Brief 8: identifying road safety hotspots, contributory factors, and evaluating
whether recent policy changes are working.

> **The main analysis is in `Final_Notebook/`.** All other folders contain
> supporting work, drafts, and documentation.

---

## Folder Details

### `Final_Notebook/`
The complete, combined analysis pipeline. Contains:
- `ST422_DataPrep.ipynb` — Step 1: cleans and merges raw STATS19 data
- `ST422_Analysis_v2.ipynb` — Step 2: full analysis (trends, hotspots, robustness, uncertainty)
- `requirements.txt` — Python dependencies
- `README.md` — full run instructions and output inventory
- `outputs/` — all figures and tables generated by the analysis

This supersedes all notebooks in `Draft_work/` and `Analysis_Practice/`.

### `Action_Log/`
Running log of all team tasks throughout the project. Each entry records
the task, owner, deadline, status, and a commit or evidence pointer.
Updated throughout the project by AS.

### `Analysis_Practice/`
Early exploratory notebooks used to familiarise the team with the STATS19
dataset before the main analysis began. **Not part of the final pipeline.**

### `Client_Resources/`
Client brief (Brief 8), DfT data documentation, and reference material
consulted during scoping and analysis.

### `Data_Prep/`
Early data preparation notebooks used before the pipeline was consolidated
into `Final_Notebook/ST422_DataPrep.ipynb`. **Superseded — do not use for
reproducibility.**

### `Draft_work/`
Draft analysis notebooks from Weeks 8–10, produced before the full pipeline
was combined into `ST422_Analysis_v2.ipynb`. Retained for audit purposes.
**Superseded — the final analysis is in `Final_Notebook/`.**

### `Management_Plan/`
Team management plan submitted as part of the consultancy portfolio. Includes
workplan with milestones, risk register, and QA plan.

### `Meeting_Minutes/`
Minutes from all team meetings, with agenda, actions agreed, owners, and
deadlines recorded. Used as evidence of group contact in the portfolio.

### `Quality_Assurance/`
QA notebooks and outputs:
- `QA_DataLoad.ipynb` — 32 automated checks on DataPrep outputs (31/32 passed)
- `README_DataPrep.md` — documents the one known issue (2,771 duplicate rows
  in `collisions_clean.csv`, confirmed not affecting `cas_full.csv`)

### `data/`
Raw DfT STATS19 CSV files. **Not committed to Git** due to file size.
Download instructions are in `Final_Notebook/README.md`.
