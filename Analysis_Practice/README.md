# ST422 Brief 8 — Road Safety Analysis

Analysis of DfT STATS19 road casualty data (2014–2025) for ST422 Statistical
Consulting. This repo contains the full data preparation and analysis pipeline
for Brief 8: identifying road safety hotspots, contributory factors, and
evaluating whether recent policy changes are working.

---

## Folder Structure

```
ST422/
├── ST422_DataPrep.ipynb              ← Step 1: data cleaning and merging
├── analysis_notebook_practice.ipynb           ← Step 2: full analysis pipeline
├── requirements.txt                  ← Python dependencies
├── README.md                         ← this file
├── Data/                             ← raw DfT STATS19 CSVs (see below)
│   └── ons_la_population_2024.csv    ← ONS mid-2024 LA population estimates
├── Cleaned/                          ← produced by ST422_DataPrep.ipynb
└── outputs/                          ← figures and tables produced by analysis
```

> `Data/` raw STATS19 CSVs and `Cleaned/` are not committed to Git — files
> are too large. `outputs/` figures and tables are regenerated by running
> the notebooks. `ons_la_population_2024.csv` is committed to `Data/`.

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Download the raw data files

Place the following files inside the `Data/` folder. Use the exact DfT
filenames — do not rename them.

```
dft-road-casualty-statistics-collision-1979-latest-published-year.csv
dft-road-casualty-statistics-casualty-1979-latest-published-year.csv
dft-road-casualty-statistics-vehicle-1979-latest-published-year.csv
dft-road-casualty-statistics-collision-provisional-2025.csv
dft-road-casualty-statistics-casualty-provisional-2025.csv
dft-road-casualty-statistics-vehicle-provisional-2025.csv
local-authority-ons-district-names.csv
ons_la_population_2024.csv
```

Download STATS19 files from:
https://www.data.gov.uk/dataset/cb7ae6f0-4be6-4935-9277-47e5ce24a11f/road-safety-data

`local-authority-ons-district-names.csv` is included in the repo under `Data/`.

`ons_la_population_2024.csv` is included in the repo under `Data/`.
Source: ONS mid-year population estimates 2024, local authority level
(Nomis, LA district/unitary as of April 2023, downloaded 19 April 2026).
Required for per-capita hotspot ranking and Poisson/NB model offset.
If this file is missing, the three-way exposure robustness check and
population-offset model are skipped gracefully — all other outputs still run.

No path editing is required. Both notebooks automatically detect their

root. As long as the folder structure above is in place, the notebooks

## Reproducibility Notes

- Always restart the kernel before a full re-run
- Run Step 1 before Step 2 — the analysis notebook reads `cas_full.csv`
  produced by the data prep notebook
- All paths are relative to project root — no local path editing needed
- No random sampling is used — all outputs are fully deterministic given
  the same input CSVs
- `collision_index` is loaded as string throughout to preserve leading zeros
- Historical files are year-filtered at load time (2014 onwards) using
  PyArrow if available, chunked pandas otherwise — full file is never
  held in memory
- Merge uses dual key `(collision_index, collision_year)` to prevent
  cross-year index collisions
- Provisional 2025 records are flagged and excluded from all primary
  analyses; used only for seasonality and annualisation checks

## Year Window

The analysis covers 2014–2024 (confirmed) and January–June 2025 (provisional).
COVID years (2020–2021) are retained in the cleaned data but excluded from
trend fitting and flagged throughout. To change the year window, edit
`YEAR_START` and `YEAR_END` in the config cell at the top of
`analysis_notebook_practice.ipynb`.

## Key Analytical Notes

**IBRS adjustment**: Raw KSI is used as the primary outcome throughout. IBRS
adoption increased substantially across the analysis window, making the adjusted
severity series inconsistent across years and unsuitable as a trend indicator.
The raw vs adjusted comparison (Fig 4) is retained for context and caveating
only. Fatalities are used as a parallel benchmark unaffected by IBRS.

**Material change threshold**: An LA is flagged if its recent 3-year average
(defined by `RECENT_YEARS`) exceeds its base 3-year average (defined by
`BASE_YEARS`) by at least `THRESHOLD_PCT`% AND its OLS slope is positive.
Red bars = CI excludes zero (statistically clear); orange = CI includes zero
(treat with caution). Adjust `THRESHOLD_PCT` in config to change sensitivity.

**Hotspot definitions**: Two definitions are compared throughout — raw KSI
count and collision-adjusted severity rate. These identify different types of
authority and imply different intervention strategies. Both are reported and
a three-way comparison including per-capita rate is produced if the population
file is available.

**Poisson/NB model**: Overdispersion is tested automatically. If Pearson
chi-squared / df > 1.5, Negative Binomial is used. Rate ratios and 95% CIs
are reported alongside OLS slope estimates as a cross-check.

**Policy evaluation**: Authorities evaluated in Section 10 are set via
`POLICY_LAS` in the config. Update this list to evaluate any authorities.
